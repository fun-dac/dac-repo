---
title: タスクの管理方法
description: タスク管理における技術的負債と返済方法について
---

## 現状

タスクは Redmine のチケットで管理している。

## 負債

- ステータスが不明のタスクが大量にあり、現状の把握が困難である
- タスクの管理方法が明文化されておらず、フォーマットに一貫性がなかったり、内容に過不足がある
- タスクのステータスやアサイニーなどが分かりにくい UI になっている

## 返済方法

- GitHub のプロジェクトで管理する
- タスクの管理方法を明文化する

## 利点

- GitHub は多くの開発者が日常的に使用しており、学習コストが低い
- PR と Issue を紐づけられるため、タスクと実装の対応関係が明確になる
- カンバン UI により、タスクのステータスやアサイニーが一目で把握できる
