# ドキュメント執筆ルール

## 構成

- 1ページにつき1つの責務に絞って書く
- 抽象から具体の順で読めるように構成する。背景・目的を先に示し、その後に具体的な手順や仕様を書く
- 内容が重複する場合はリンクで参照し、同じ内容を複数ページに書かない
- ページを追加した場合は `apps/docs/content/docs/meta.json` を編集する
- サイドバーはなるべくフラットに保ち、不要なネストを避ける

## 文体

- 体言止めは使わない
- カジュアルな表現を避け、「である」調で統一する
- 用語は表記を統一し、同じ概念に異なる表記を使わない
